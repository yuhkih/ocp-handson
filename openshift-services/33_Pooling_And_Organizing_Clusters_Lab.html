Pooling and Organizing Clusters Lab

1. Introduction to Pooling and Organizing Clusters

Red Hat® OpenShift® Container Platform クラスターのフリートを管理するには、多くのクラスターのプロビジョニングと破棄が必要になることがよくあります。 ご覧のとおり、新しいクラスターが作成されるまでには長い時間がかかります。

このラボでは、クラスター プールとクラスター セットの使用方法を学習します。

Cluster pools は、OpenShift クラスターをプロビジョニングするために使用されます。このクラスターは休止状態になり、要求されるのを待ちます。 クラスターの要求は、休止状態のクラスターをクラスター プールから再開するのと同じくらい迅速です。 クラスタープール内のクラスターが要求されると、Red Hat Advanced Cluster Management for Kubernetes (RHACM) は新しいクラスターを自動的にデプロイしてプールを補充します。

クラスター セット は、クラスターを整理するのに役立ちます。たとえば、ロールベースのアクセス制御 (RBAC) を管理して、運用クラスターに監査者のアクセス許可を設定します。

Goals

  • クラスター プールを作成し、要求されるクラスターを準備する

  • 異なるクラスター セットでクラスターを編成する

  • RBAC 権限を管理する

  • OpenShift のバージョンを管理する

2. Explore Cluster Pools

このセクションでは、クラスター プールを使用してクラスターを管理します。

2.1. Create Cluster Set

クラスタ プールを作成する前に、プールで使用する「ClusterSet」を作成する必要があります。

手順

  1. 現在 RHACM コンソールにログインしていない場合は、再度ログインします。

  2. Infrastructure → Clusters に移動します。

  3. Cluster sets タブを選択します。

  4. Create cluster set をクリックします。

  5. Cluster set name として production (小文字に注意) を使用し、Create をクリックします。

  6. 表示されるダイアログ ボックスで、Close をクリックします。

2.2. Create production Cluster Pool

クラスタ セットが利用可能になったので、クラスタ プールを作成できます。

クラスター プールの作成プロセスは、スタンドアロン クラスターの作成と非常によく似ています。 2 つの異なる地理的リージョンに単一ノードの OpenShift クラスターのプールを作成します。

手順

  1. Infrastructure → Clusters に移動します。

  2. Cluster pools タブを選択します。

  3. Create cluster pool をクリックします (ボタンを表示するには、下にスクロールする必要がある場合があります)。

  4. 次のように、最初のクラスター プールのプロパティを入力します。

    1. Infrastructure provider 画面で以下のように入力します。

      • Infrastructure provider: Amazon Web Services

      • リストから aws-sandbox 資格情報を選択します。

    2. Next をクリックします。

    3. Cluster pool details 画面で以下のように入力します。

      • Cluster pool name: production-uk

      • Cluster pool namespace: cluster-pool-production-uk

      • Cluster pool size: 1

      • Cluster set: production

      • Release image: 利用可能な OpenShift 4.9 イメージを選択します (例: 4.9.23)。

    4. Next をクリックします。

    5. Node pools 画面で以下のように入力します。

      • Region: eu-west-2

      • Control plane pool: セクションを展開し、Instance type を m5.2xlarge に設定します。

      • Worker pool 1: セクションを展開し、Node count を 0 に設定します。

    6. Next をクリックします。

    7. Networking 画面で以下のように入力します。

      • Cluster network CIDR: 10.132.0.0/14 (変更前のDefault表示: 10.128.0.0/14)

      • Service network CIDR: 172.31.0.0/16 (変更前のDefault表示: 172.30.0.0/16)

    8. Review 画面が表示されるまで Next をクリックします。

    9. Review 画面で以下のように入力します。

      • YAML を On に設定します。

      • Cluster YAML ペインの ClusterPool タブを選択し(初期状態で選択されています)、kind: ClusterPool を見つけて、数行下の spec: の次の行に " skipMachinePools: true" (行頭に空白２個あるのでご注意ください)フィールドを加えます。

      • install-config タブに切り替えて controlPlane を見つけ、その下の replicas フィールドを 1 に変更します。

    10. Create をクリックします。

  5. 最初のクラスター プールの作成中に、Create cluster pool をクリックして 2 つ目のクラスター プールを作成します。

  6. 次のように、2 番目のクラスター プールのプロパティを入力します。

    1. Infrastructure provider 画面で以下のように入力します。

      • Infrastructure provider: Amazon Web Services

      • リストから aws-sandbox 資格情報を選択します。

    2. Next をクリックします。

    3. Cluster pool details 画面で以下のように入力します。

      • Cluster pool name: production-ireland

      • Cluster pool namespace: cluster-pool-production-ireland

      • Cluster pool size: 1

      • Cluster set: production

      • Release image: 利用可能な OpenShift 4.9 イメージを選択します (例: 4.9.23)。

    4. Next をクリックします。

    5. Node pools 画面で以下のように入力します。

      • Region: eu-west-1

      • Control plane pool: セクションを展開し、 Instance type を m5.2xlarge に設定します。

      • Worker pool 1: セクションを展開し、Node count を 0 に設定します。

    6. Next をクリックします。

    7. Networking 画面で以下のように入力します。

      • Cluster network CIDR: 10.132.0.0/14 (変更前のDefault表示: 10.128.0.0/14)

      • Service network CIDR: 172.31.0.0/16 (変更前のDefault表示: 172.30.0.0/16)

    8. Review 画面が表示されるまで Next をクリックします。

    9. Review 画面で以下のように入力します。

      • YAML を On に設定します。

      • Cluster YAML ペインの ClusterPool タブを選択し(初期状態で選択されています)、kind: ClusterPool を見つけて、数行下の spec: の次の行に " skipMachinePools: true" (行頭に空白２個あるのでご注意ください)フィールドを加えます。

      • install-config タブに切り替えて controlPlane を見つけ、その下の replicas フィールドを 1 に変更します。

    10. Create をクリックします。

2.3. Scale Cluster Pool

ロンドン (リージョン eu-west-2 ) のクラスター プールがアイルランドのクラスター プールよりも頻繁に使用されることが予想されるため、クラスター プールを 2 つのレプリカにスケーリングします。

手順

  1. Infrastructure → Clusters に移動します。

  2. Cluster pools タブを選択します。

  3. 2 つのクラスター プールを展開します。

  4. (Options) をクリックし、Scale cluster pool を選択して、production-uk クラスター プールをスケールします。

  5. Desired pool size を 2 に設定し、Scale をクリックします。

  6. すべてのクラスターがプロビジョニングされるまで待ちます。

    クラスターのプロビジョニングには、約 30 ～ 40 分かかります。 クラスターがプロビジョニングされると、休止状態(Hybernate)になります。

  7. プールされた各クラスターの横にある Creating をクリックし、View logs を選択してクラスター作成ログを追跡します。

    プール内のすべてのクラスターは、クラスター プールの作成時に選択したクラスター セットに属しています。このシナリオでは、すべてのクラスターは production クラスター セットに属しています。

3. Summary

このラボでは、クラスタ セットを使用してクラスタをプールし、クラスタをグループ化する機能について学習しました。

クラスター プールを使用すると、休止状態のクラスターのプールから新しいクラスターを要求するのに必要な時間が短縮され、要求されるスタンバイ クラスターのホスティング コストも削減されることを学びました。

クラスター セットを使用して OpenShift クラスターの同一のセットをデプロイし、バージョン、ノード数などのさまざまな OpenShift 構成をより簡単に管理できるようにしました。
